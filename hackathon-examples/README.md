# Cluster Creation Examples

Verified end-to-end against **cloud-dev** in project `688352869c0da273e13426a6`
using the Atlas Admin API `2024-10-23` (Independent Shard Scaling format — no
`numShards`). Each request was submitted with the autogenerated `atlas api`
command group, which is a thin 1:1 wrapper over the Admin API:

```bash
atlas api clusters createCluster \
  --profile default \
  --version 2024-10-23 \
  --groupId <projectId> \
  --file <request>.json \
  --pretty
```

Both examples enable **compute auto-scaling** (with scale-down) and **disk
auto-scaling** per region config. The `autoScaling` block goes on each
`regionConfigs[]` entry — set it on every shard/region where you want scaling
to apply.

Endpoint: `POST /api/atlas/v2/groups/{groupId}/clusters`

| File | Shape |
| --- | --- |
| [replica-set-request.json](./replica-set-request.json) | `REPLICASET`, AWS US_EAST_1, 3×M10, autoscale M10→M40 |
| [replica-set-response.json](./replica-set-response.json) | Full API response (`stateName: CREATING`) |
| [sharded-request.json](./sharded-request.json) | `SHARDED`, 2 shards × 3×M30 in AWS US_EAST_1, autoscale M30→M60 |
| [sharded-response.json](./sharded-response.json) | Full API response (`stateName: CREATING`) |

## Replica set — request highlights

```json
{
  "name": "example-replica-set",
  "clusterType": "REPLICASET",
  "replicationSpecs": [{
    "zoneName": "Zone 1",
    "regionConfigs": [{
      "providerName": "AWS",
      "regionName": "US_EAST_1",
      "priority": 7,
      "electableSpecs": { "instanceSize": "M10", "nodeCount": 3 },
      "autoScaling": {
        "compute": {
          "enabled": true,
          "scaleDownEnabled": true,
          "minInstanceSize": "M10",
          "maxInstanceSize": "M40"
        },
        "diskGB": { "enabled": true }
      }
    }]
  }]
}
```

## Sharded — request highlights

In the `2024-10-23` API each shard is its own entry in `replicationSpecs[]` —
that's what unlocks asymmetric shard sizing. For a symmetric two-shard cluster
the two entries are identical:

```json
{
  "name": "example-sharded",
  "clusterType": "SHARDED",
  "replicationSpecs": [
    { "zoneName": "Zone 1", "regionConfigs": [ /* shard 1: 3×M30 + autoscale */ ] },
    { "zoneName": "Zone 1", "regionConfigs": [ /* shard 2: 3×M30 + autoscale */ ] }
  ]
}
```

## Notes

- `priority: 7` on the sole region config makes it the primary region (required
  for dedicated tiers).
- When `compute.enabled` is `true`, `maxInstanceSize` is required; with
  `scaleDownEnabled: true` you must also set `minInstanceSize`.
- Instance sizes in the response reflect the **current** size; autoscaling may
  move them inside `[min, max]` over time.
- Regenerate the responses any time by re-running the two `atlas api` commands
  above — they are idempotent per cluster name.
